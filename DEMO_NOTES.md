# Demo Notes

These notes help you capture screenshots or run a client demo in a clean order.

## Suggested Screenshot Set

1. Landing page or main workspace.
2. Settings page showing backend status and OCR readiness.
3. Knowledge Base page with documents loaded.
4. OCR page with file upload and processing mode selection.
5. OCR result card showing saved output.
6. Chat or writing tool showing the knowledge base in use.

## Suggested Demo Script

1. Open the app and show the main writing tools.
2. Open Settings and point out the backend and Deep Extract status.
3. Open Knowledge Base and import a sample PDF or folder.
4. Open OCR and process a PDF or image.
5. Save the OCR output into the Knowledge Base.
6. Show that the UI remains responsive while jobs run.

## Talking Points

- The app keeps user data local by default.
- OCR can run in browser mode or through the local Python backend.
- Deep Extract is optional and only used when MinerU is installed.
- Knowledge Base import supports local content and structured reasoning.

## File Checklist For Screenshots

- Use a small PDF for a quick demo
- Use a folder with a few text files if you want to show bulk ingestion
- Keep API keys hidden in screenshots
- Close unrelated tabs to reduce visual noise
